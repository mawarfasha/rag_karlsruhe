# GeoRAG – Geo-aware RAG System for Karlsruhe

A **Retrieval-Augmented Generation (RAG)** system that answers questions about
restaurants, cafés, tourist attractions, and shops in **Karlsruhe, Germany**.

It fetches real data from **OpenStreetMap**, stores place descriptions as
vectors in **Milvus**, and generates natural-language answers using **Mistral**
(via Ollama). An interactive route map is generated by default for each query
(disable with `--no-map` if needed).

---

## System Architecture

```
User Question
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│                    GeoRAG Pipeline                          │
│                                                             │
│  1. Embed question  ──→  all-MiniLM-L6-v2  (384-dim)       │
│  2. Search Milvus   ──→  top-5 nearest place vectors        │
│  3. Build context   ──→  place descriptions + metadata      │
│  4. Generate answer ──→  Mistral 7B (via Ollama)            │
└─────────────────────────────────────────────────────────────┘
      │                          │
      ▼                          ▼
 Text Answer               Route Map (HTML)
                        (opens in browser)
```

### Components

| Component | Technology | Purpose |
|-----------|-----------|---------|
| Data source | OpenStreetMap / Overpass API | Free real-world geo data |
| Embedding model | `all-MiniLM-L6-v2` (Sentence Transformers) | Convert text → 384-dim vectors |
| Vector database | **Milvus Standalone (Docker)** | Fast semantic search over stored place vectors |
| RAG framework | **LlamaIndex** | Document pipeline & query engine |
| LLM | **Mistral 7B** via **Ollama** | Generate natural-language answers |
| Map visualisation | **Folium** | Interactive HTML route maps |
| CLI | argparse | User-friendly command-line interface |

## Installation

### 1. Create a virtual environment (recommended)

```bash
# In the georag/ folder:
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

### 2. Install Python packages

```bash
pip install -r requirements.txt
```

> **Note – PyTorch on Windows CPU:**  
> If `pip install torch` is slow or fails, install the CPU-only version directly:
> ```bash
> pip install torch --index-url https://download.pytorch.org/whl/cpu
> ```

### 3. Start Milvus (Docker)

```bash
docker compose up -d
```

Milvus API endpoint used by this project: `http://localhost:19530`

### 4. Install Ollama and pull Mistral (for AI responses)

```bash
# 1. Download and install Ollama from https://ollama.com/download
# 2. Start the Ollama server (leave this terminal open):
ollama serve
# 3. In a second terminal, download the Mistral model (~4 GB):
ollama pull mistral
```

> **Without Ollama:** The system still works – it returns the retrieved
> places formatted as plain text.  You only miss the AI-generated sentences.

## Quick Start

### Step 1 – Ingest OpenStreetMap data  *(run once)*

```bash
python main.py ingest
```

This will:
- Fetch restaurants, cafés, attractions, and shops in Karlsruhe via the Overpass API
- Generate embeddings for each place using `all-MiniLM-L6-v2`
- Store everything in the Milvus Docker collection `karlsruhe_places`

Takes about **2–5 minutes** on first run (model download + embedding generation).

### Step 2 – Ask a question

```bash
python main.py ask "Where can I find good sushi near Europaplatz?"
```

An interactive HTML map is automatically saved to `outputs/route_map.html`.
Open it in any web browser to see the suggested route.

---

## Usage

### `ingest` – Fetch and index OSM data

```bash
python main.py ingest              # use cached data if available
python main.py ingest --fresh      # force re-fetch from OpenStreetMap
```

### `ask` – Query the system

```bash
# Basic question (uses Marktplatz as default location)
python main.py ask "Where can I bring my kids today?"

# Specify your GPS location
python main.py ask "What cafés are near me?" --location "49.0069,8.4037"

# Also show the retrieved context documents
python main.py ask "Where to buy a washing machine?" --show-sources

# Save the full output as JSON
python main.py ask "Best Italian restaurant?" --save-output outputs/q1.json

# Skip map generation
python main.py ask "Quick coffee?" --no-map
```

### `check` – Verify your setup

```bash
python main.py check
```

### Run all 5 example queries

```bash
python examples/example_queries.py
```

## Configuration (`config.py`)

| Setting | Default | Description |
|---------|---------|-------------|
| `EMBEDDING_MODEL` | `all-MiniLM-L6-v2` | Sentence Transformers model |
| `EMBEDDING_DIM` | `384` | Vector dimension |
| `LLM_MODEL` | `mistral` | Ollama model name |
| `CHUNK_SIZE` | `512` | Token chunk size |
| `CHUNK_OVERLAP` | `50` | Token overlap between chunks |
| `TOP_K` | `5` | Number of places to retrieve per query |
| `KARLSRUHE_BBOX` | `48.97…,8.33…` | Geographic search boundary |

## Troubleshooting

**"No database found" error:**
```bash
python main.py ingest
```

**Ollama not responding:**  
Make sure the server is running: `ollama serve`

**Slow embedding on first run:**  
The model is downloaded from HuggingFace Hub (~90 MB) and cached.
Subsequent runs are fast.

**`torch` installation issues on Windows:**
```bash
pip install torch --index-url https://download.pytorch.org/whl/cpu
```

**Overpass API timeout:**  
The public API can be slow. The fetched data is cached in `data/places.json`
after the first successful run, so subsequent `ingest` calls are instant.

## How RAG Works (Project Version)

1. **Ingestion:** Every place's description (e.g. *"Sushi Circle is a restaurant serving sushi cuisine, located at Kaiserstraße 12, Karlsruhe"*) is converted to a 384-number vector by the embedding model and stored in Milvus.

2. **Query:** When you ask *"best sushi near Europaplatz?"*, your question is also converted to a vector.  Milvus finds the 5 place vectors that are mathematically closest (most similar in meaning).

2.5 **Reranking:** Retrieved candidates are reranked using a hybrid score that combines semantic relevance and distance from your location.

3. **Generation:** The 5 retrieved place descriptions are passed to Mistral as context, together with your question. The model then generates the final answer from that context.
